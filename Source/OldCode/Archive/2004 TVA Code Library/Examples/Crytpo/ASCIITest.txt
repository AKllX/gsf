<h2><a name="_Toc54603496"></a><a name="_Toc52603913"><span style='mso-bookmark:
_Toc54603496'>TDES System Description</span></a><o:p></o:p></h2>

<p class=MsoNormal>&nbsp;<o:p></o:p></p>

<p class=MsoNormal>The Telegyr Data Exchange Service or TDES acts as a general
data and messaging exchange system between the TVA network (referenced as the
public network) and the Telegyr network (referenced as the private
network).&nbsp; The system consists of two separate services that communicate
using XML and binary packets sent over FTP.&nbsp; For security reasons, this
will be the only available communications channel that exists between the two
networks once the DMZ is operational (see <a
href="#_Telegyr_7.3_Data_Exchange Service A">TDES diagram</a>).&nbsp; The
system handles the following functions:<o:p></o:p></p>

<p class=MsoNormal>&nbsp;<o:p></o:p></p>

<p class=MsoNormal>1)&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Allows bulk dynamic <span
class=GramE>inserts,</span> updates and deletions of the SCADA source database that
can be pushed to live Telegyr system<o:p></o:p></p>

<p class=MsoNormal>2)&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Daily and on-demand SCADA
source database replication  data used by TDES, NERC Report and Point Report.<span
style='mso-spacerun:yes'>  </span>Process steps:<o:p></o:p></p>

<p class=MsoNormal style='margin-left:.5in;text-indent:.5in'>a) &nbsp;Export
all data from SDB<o:p></o:p></p>

<p class=MsoNormal style='margin-left:.5in;text-indent:.5in'>b) &nbsp;FTP
exported data to public system<o:p></o:p></p>

<p class=MsoNormal style='margin-left:.5in;text-indent:.5in'>c)&nbsp; Backup
export file that can be used for offline data updates<o:p></o:p></p>

<p class=MsoNormal style='margin-left:.5in;text-indent:.5in'>d) &nbsp;Import
SDB replica into OPDATSQL<span class=GramE>:SDB2</span><o:p></o:p></p>

<p class=MsoNormal>3)&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Monitors scheduled
SQL server backups of the SCADA source database</p>

<p class=MsoNormal>4)&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Acts as an equipment
ping proxy for systems that will no longer be <span class=SpellE>pingable</span>
once DMZ is in production<o:p></o:p></p>

<p class=MsoNormal>5)&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; System health/status
monitoring (both Private and Public)<o:p></o:p></p>

<p class=MsoNormal>6)&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Provides a general
XML based communications mechanism between systems isolated by the DMZ<o:p></o:p></p>

<p class=MsoNormal>7)&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Provides on-demand
status information to web pages needing current service state<o:p></o:p></p>

<p class=MsoNormal>&nbsp;<o:p></o:p></p>

<p class=MsoNormal>The public side (TVA network) of the TDES service is
installed on chasocopctrl2 at C:\Program Files\TVA\<span class=SpellE>PublicTDES</span>.&nbsp;
The service is named <b style='mso-bidi-font-weight:normal'>Telegyr Data
Exchange Service (Public)</b> should it ever need to be restarted.<o:p></o:p></p>

<p class=MsoNormal>&nbsp;<o:p></o:p></p>

<p class=MsoNormal>The private side (Telegyr network) of the TDES service is
installed on SDB (the SCADA source database SQL Server box located in the SOC
server room) at G:\Program Files\TVA\<span class=SpellE>PrivateTDES</span>.&nbsp;
The service is named <b style='mso-bidi-font-weight:normal'>Telegyr Data
Exchange Service (Private)</b> should it ever need to be restarted.<o:p></o:p></p>

<p class=MsoNormal>&nbsp;<o:p></o:p></p>

<p class=MsoNormal>The public side of the TDES system reports to SSAM (see <a
href="#_System_Detail_as_Defined in SSAM">SSAM diagram</a>) on the health of
the public service, the health of the private service, the health of the DMZ
transfer service (the DMZ transfer app not yet in production), full cycle ping
between systems, daily export data flow from SDB, daily import data flow into
OPDATSQL, monitors scheduled SDB SQL Server backup successes and failures, and
acts as an equipment ping proxy for SDB, TVAA, TVAB and TVAEBS since these boxes
will no longer be <span class=SpellE>pingable</span> once the DMZ is in
place.<o:p></o:p></p>

<p class=MsoNormal>&nbsp;<o:p></o:p></p>

<h2><a name="_Toc54603497">Shared Workspace Template File Definitions</a></h2>

<p class=MsoNormal align=right style='text-align:right'><b style='mso-bidi-font-weight:
normal'><a href="javascript:parent.window.close()">Close this window</a><o:p></o:p></b></p>

<p class=MsoNormal><b style='mso-bidi-font-weight:normal'><o:p>&nbsp;</o:p></b></p>

<p class=MsoNormal>The following files are the templates used by the TDES
system for performing bulk data operations.<span style='mso-spacerun:yes'> 
</span>Be careful not to unintentionally modify these files (see <a
href="#_Updating_the_Shared_Template Files"><span class=GramE>Updating</span>

<h3><a name="_Toc54603503">Key Notes</a></h3>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

to the Telegyr Source Database there are <b style='mso-bidi-font-weight:normal'>no
restrictions</b> to what fields or tables you can modify, be it for Inserts,
Updates or Deletes.<span style='mso-spacerun:yes'>  </span>In order to properly
make any kind of general database modifications you must have a working
understanding of the SDB <span class=SpellE><b style='mso-bidi-font-weight:
normal'>TG_Source</b></span> SQL Server database structure.<span
style='mso-spacerun:yes'>  </span>Before jumping into this headlong I would
recommend reading the <a
href="../../../../CtrlSys/Telegyr/TelegyrDataExchangeService/SDBDocumentation.html#ProgramDataDefinition"
target="_blank">Program Data Definition</a> section of the SDB Documentation.</p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

 this table is where all the <b style='mso-bidi-font-weight:normal'><i
style='mso-bidi-font-style:normal'>Unique Identifiers</i></b> (or <span
class=SpellE><b style='mso-bidi-font-weight:normal'>UID</b>s</span>) for every
entity in the system are stored and generated:</p>

<h3><a name="_Toc54603504">Steps for Performing Data Inserts</a></h3>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>1) Copy a new version of <b style='mso-bidi-font-weight:
normal'>SDBUpdate.mdb</b> anywhere onto your local system  for reference
purposes you can rename this file to anything you like.<span
style='mso-spacerun:yes'>  </span>This file is your isolated <b
style='mso-bidi-font-weight:normal'><i style='mso-bidi-font-style:normal'>Workspace</i></b>
for making data updates (if you want to update the shared template, click <b
style='mso-bidi-font-weight:normal'><a
href="#_Updating_the_Shared_Template_Files">here</a></b>). <span
style='font-size:9.0pt'><o:p></o:p></span></p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>2) Open the <b style='mso-bidi-font-weight:normal'>SDBUpdate.mdb</b>
using <b style='mso-bidi-font-weight:normal'>Microsoft Access</b></p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>3) Begin inserting desired records, note that all data you
desire to insert should end up in the <b style='mso-bidi-font-weight:normal'><i
style='mso-bidi-font-style:normal'>non-linked</i></b> tables in this file</p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>4) Once you are satisfied with your changes, go back to the
main web page for the <b style='mso-bidi-font-weight:normal'><i
style='mso-bidi-font-style:normal'>Telegyr 7.3 Data Exchange Service</i></b>
and click <b style='mso-bidi-font-weight:normal'>Browse</b> under the <b
style='mso-bidi-font-weight:normal'>Upload Workspace</b> section.<span
style='mso-spacerun:yes'>  </span>This will bring up a file selection dialog
where you should select the <b style='mso-bidi-font-weight:normal'>SDBUpdate.mdb</b>
you have been modifying.</p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>5) Set the <b style='mso-bidi-font-weight:normal'>Workspace
Type</b> to <b style='mso-bidi-font-weight:normal'><i style='mso-bidi-font-style:
normal'>Structured Access Database</i></b> and the <b style='mso-bidi-font-weight:
normal'>Update Type</b> to <b style='mso-bidi-font-weight:normal'><i
style='mso-bidi-font-style:normal'>Adding Records</i></b>.</p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>6) When you have set your options and selected your file,

<h3><a name="_Toc54603505">Steps for Performing Data Updates</a></h3>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>1) Copy a new version of <b style='mso-bidi-font-weight:
normal'>SDBUpdate.mdb</b> anywhere onto your local system  for reference
purposes you can rename this file to anything you like.<span
style='mso-spacerun:yes'>  </span>This file is your isolated <b
style='mso-bidi-font-weight:normal'><i style='mso-bidi-font-style:normal'>Workspace</i></b>
for making data updates (if you want to update the shared template, click <b
style='mso-bidi-font-weight:normal'><a
href="#_Updating_the_Shared_Template_Files">here</a></b>). <span
style='font-size:9.0pt'><o:p></o:p></span></p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>2) Open the <b style='mso-bidi-font-weight:normal'>SDBUpdate.mdb</b>
using <b style='mso-bidi-font-weight:normal'>Microsoft Access</b>, note that
all data you desire to update should end up in the <b style='mso-bidi-font-weight:
normal'><i style='mso-bidi-font-style:normal'>non-linked</i></b> tables in this
file.</p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>3) Write simple queries to locate the desired records to
update (queries should be written to access <b style='mso-bidi-font-weight:
normal'><i style='mso-bidi-font-style:normal'>linked table data</i></b>, i.e.,
tables suffixed with the number <b style='mso-bidi-font-weight:normal'>1</b>).</p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal><span style='mso-tab-count:1'>          </span>For example,
the following selects all the <b style='mso-bidi-font-weight:normal'>STATUS</b>
measurements from the substation named <st1:City><st1:place><b
  style='mso-bidi-font-weight:normal'>ABERDEEN</b></st1:place></st1:City>:</p>

<p class=MsoNormal style='text-indent:.75in'><o:p>&nbsp;</o:p></p>

<p class=MsoNormal style='text-indent:.75in'><b style='mso-bidi-font-weight:
normal'>Design View:<o:p></o:p></b></p>

<p class=MsoNormal style='text-indent:1.0in'><!--[if gte vml 1]><v:shape id="_x0000_i1028"
 type="#_x0000_t75" style='width:347.25pt;height:285.75pt'>
 <v:imagedata src="ReadMe_files/image013.png" o:title=""/>
</v:shape><![endif]--><![if !vml]><img border=0 width=463 height=381
src="ReadMe_files/image014.jpg" v:shapes="_x0000_i1028"><![endif]></p>

<p class=MsoNormal style='text-indent:.75in'><o:p>&nbsp;</o:p></p>

<p class=MsoNormal style='text-indent:.75in'><b style='mso-bidi-font-weight:
normal'>SQL View:<o:p></o:p></b></p>

<p class=MsoNormal style='text-indent:1.0in'><span style='font-size:10.0pt;
font-family:"Courier New"'>SELECT Measurement1.*<o:p></o:p></span></p>

<p class=MsoNormal style='text-indent:1.0in'><span style='font-size:10.0pt;
font-family:"Courier New"'>FROM Measurement1 INNER JOIN Substation1 ON
Measurement1.Substation = Substation1.UID<o:p></o:p></span></p>

<p class=MsoNormal style='text-indent:1.0in'><span style='font-size:10.0pt;
font-family:"Courier New"'>WHERE Measurement1.MeasurementClass='STATUS' AND
Substation1.TG8000_Name='ABERDEEN'<o:p></o:p></span></p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>4) When your query successfully selects the records which
you would like to update, select all of the records (select the <b
style='mso-bidi-font-weight:normal'>Edit</b> menu and click <b
style='mso-bidi-font-weight:normal'>Select All Records</b>), then copy the
records (select the <b style='mso-bidi-font-weight:normal'>Edit</b> menu and
click <b style='mso-bidi-font-weight:normal'>Copy</b>).</p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>5) Now open the destination table (in the above example,
this would be <b style='mso-bidi-font-weight:normal'>Measurement</b>  the <b

table) and paste the copied records (select the <b style='mso-bidi-font-weight:

Append</b>).</p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>6) Now you have a working copy of the records you would like
to update.<span style='mso-spacerun:yes'>  </span>You can now make changes to
this data as needed using update queries or manual edits.</p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal><span style='mso-tab-count:1'>          </span>For example,
the following would change the <span class=SpellE><b style='mso-bidi-font-weight:

style='mso-bidi-font-weight:normal'>Measurement</b> records:</p>

<p class=MsoNormal style='text-indent:.75in'><o:p>&nbsp;</o:p></p>

<p class=MsoNormal style='text-indent:.75in'><b style='mso-bidi-font-weight:
normal'>SQL View:<o:p></o:p></b></p>

<p class=MsoNormal style='text-indent:1.0in'><span style='font-size:10.0pt;
font-family:"Courier New"'>UPDATE Measurement SET <span class=SpellE>AOR_Group</span>
= 'NWEST' WHERE <span class=SpellE>AOR_Group</span> = 'SWEST'<o:p></o:p></span></p>

<p class=MsoNormal style='text-indent:1.0in'><span style='font-size:10.0pt;

<p class=MsoNormal>7) Once you are satisfied with your changes, go back to the

normal'>Update Type</b> to <b style='mso-bidi-font-weight:normal'><i
style='mso-bidi-font-style:normal'>Updating Records</i></b>.</p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>9) When you have set your options and selected your file,
click the <b style='mso-bidi-font-weight:normal'>Upload</b> button.</p>

<p class=MsoNormal>10) When the process has completed you can go into SDB and click
