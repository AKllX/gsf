Subject: Interesting note about .NET exception handling using the "Finally" keyword:

 In the following code you might have expected that Finally clause might not run since you requested to exit the function in the Catch clause.  However, the Finally code does execute – still after the Catch but before the function exits – very nice!

 

    Public Function FinallyTest() As Boolean

 

        Try

            Throw New InvalidOperationException("Test exception...")

        Catch ex As Exception

            Debug.WriteLine("Executing Exception handler...")

            Return False

        Finally

            Debug.WriteLine("Executing Finally handler...")

        End Try

 

        Return True

 

    End Function

 

DEBUG OUTPUT:

 

Executing Exception handler...

Executing Finally handler...

False

The program '[3920] FinallyTest.exe' has exited with code 0 (0x0).
