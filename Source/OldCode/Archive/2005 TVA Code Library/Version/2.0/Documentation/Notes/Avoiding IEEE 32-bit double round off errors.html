Found this round-off issue when using Math.Floor (note Math.Floor is not the problem – just round off constraints using IEEE 32-bit double):

 

   500 / (1000 / 30) = 15 (calculator will show round-off error: 15.000000000000000000000000000002)

   Using doubles  (32-bit floats): Math.Floor(500# / (1000# / 30#)) = 14  <- This is a bug!

   Using decimals (64-bit floats): Math.Floor(500@ / (1000@ / 30@)) = 15  <- This is correct.

 

Bottom line: when it matters use Decimal values where round-off errors may be an issue...
