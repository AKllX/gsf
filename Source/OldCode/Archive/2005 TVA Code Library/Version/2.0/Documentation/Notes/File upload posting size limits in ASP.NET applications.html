If you are setting up a web service in IIS and expect to receive a big message *or* you are uploading large files to your web application, make sure you add the following entry to your ASP.NET application’s Web.config file:

 

Under the <system.web> section:

 

<httpRuntime maxRequestLength="102400" executionTimeout="600" />

 

By default this configuration setting is missing and defaults to what’s in the .NET system.config file – a default installation sets this to a maximum post size of 4MB.  The above example increases this to 100MB and sets an execution timeout of 10 minutes…
