Subject: Reseeding IDENTITY column in SQL server...

After you have deleted all the records from a table (e.g., DELETE FROM ProductPricing), the identity counter value will still be sitting as its last increment value.  To reset this you can issue the following command:

 

DBCC CHECKIDENT (ProductPricing, RESEED, 0)

GO

 

Note that if you use a TRUNCATE TABLE the counter value will automatically be reset as well – but you may not have rights to issue a TRUNCATE TABLE or you may have referential constraints preventing its execution – in these cases you can still use the above command…
